## 1) Test Taxonomy

### 1.1 Unit Tests (fast, <100ms)

**Scope**
- Header parsing (Accept-Encoding, Range, Content-Length consistency).
- Capability token envelope parsing (structure only; no crypto semantics here).
- Config parsing/validation (caps/timeouts/â€œdanger_okâ€ guards).

**Location**

**Key invariants**

### 1.2 Integration Tests (end-to-end)

**Scope**

* Full HTTP ingress â†’ handlers â†’ (mocked) downstream calls.
* Readiness lifecycle, backpressure, and degrade-first behavior.
* Error taxonomy and JSON error body shape (stable fields).
* Config reload semantics (SIGHUP/`/config/reload` if applicable).

**Location**

* `tests/*.rs`

**Must cover**

1. **Control endpoints**

   * `/healthz` â†’ 200 (process alive).
   * `/readyz` â†’ 503 until bus/deps ready; 200 after.
   * `/metrics` exposes golden metrics.
2. **Routing & Happy paths**

   * `GET /o/{addr}`: 200 with small bodies (16/64/256 KiB).
   * Range requests: valid `206` for satisfiable; `416` on invalid ranges.
3. **Safety caps**

   * Body > 1 MiB â†’ `413 body-cap` with structured JSON (`reason`, `cap_bytes`).
   * Encoded body exceeding decode ratio (e.g., gz ~12 MiB from 100 KiB) â†’ `413 decoded-cap`.
4. **Backpressure & Quotas**

   * Spike to 600 rps at default caps â†’ early 429 with `Retry-After` header; sustained reject rate <1% once load returns to steady-state.
5. **Error taxonomy**

   * 400 double `Content-Length`.
   * 401/403 capability missing/invalid.
   * 405 wrong method to route.
   * 411 missing length where required.
   * 415 unsupported `Content-Encoding`.
   * 416 unsatisfiable ranges.
   * 503 when `/readyz` not ready or degrading due to downstream.
6. **Config reload**

   * Flip rate limit and inflight caps; verify new limits applied without process restart.
7. **Degrade-first**

   * Under induced downstream slowness, writes shed (429/503) while reads continue, then `/readyz` flips to 503 when guardrails engage.

**Run**

```bash
cargo test -p svc-gateway --test '*'
```

## 7) Canonical Scenarios (Must Not Regress)

1. **Body overflow**: 1,048,577-byte POST â†’ 413 `body-cap`; memory bounded.
2. **Decode overflow**: 100 KiB gz â†’ ~12 MiB decoded â†’ 413 `decoded-cap`; process steady.
3. **Burst ingress**: 600 rps for 60s â†’ early 429; p95 recovers < 2Ã— baseline within 30s after spike.
4. **Downstream stall**: overlay/index 500 ms P99 â†’ `/readyz` degrades; writes shed; reads continue â‰¤ target tails.
5. **Ambiguous length**: both `Content-Length` and `Transfer-Encoding: chunked` â†’ 400.
6. **Range edge**: `Range: bytes=-1` on 0-length body â†’ 416; no panic.
7. **Config reload**: halve rate limit; reject curve tracks new ceiling within 1s buckets.
8. **Loom drain**: cancel ~25% of requests mid-flight â†’ no deadlocks, resources freed.
